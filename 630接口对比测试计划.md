## 四、技术方案

### 4.1 新增 Controller

在 `mock-knowledge-service` 下新建 `CompareController.java`：

```
POST /api/compare/enterprise
  → 对指定企业，调通所有有权限的630接口
  → 返回每条接口的响应数据 + 与科大讯飞数据的差异
```

### 4.2 核心流程

```
1. 接收请求 { enterpriseName, nameType, interfaces[] }
2. 并行调用生产环境 https://hub.amardata.com/credit/api/gateway
3. 对每个接口：
   a. 用 POC 账号 + 企业名 构造请求体
   b. 调接口获取实时数据
   c. 从科大讯飞 xlsx 中读取对应企业的离线数据
   d. 逐字段对比（一级字段 + JSON展开）
   e. 记录差异：仅实时有、仅离线有、值不一致
4. 汇总输出对比报告
```

### 4.3 新增文件清单

| 文件 | 说明 |
|------|------|
| `CompareController.java` | 对比接口入口 |
| `CompareService.java` | 对比逻辑：调630接口 + 读xlsx + 对比 |
| `CompareRequest.java` | 请求模型 |
| `CompareResponse.java` | 响应模型（包含差异明细） |
| `InterfaceConfig.java` | 接口配置：transcode → params模板 |
| `XlsxReader.java` | 读取科大讯飞 xlsx 的工具类 |

### 4.4 依赖

- `java.net.http.HttpClient`（JDK11自带）调630接口
- `openpyxl` 的数据已提前转换为 JSON（放在 offline-data 目录下）
- 不新增 Maven 依赖

## 五、对比维度

| 维度 | 说明 |
|------|------|
| 字段存在性 | 实时有 vs 离线有 / 一方缺失 |
| 字段值一致性 | 字符串/数字/日期 精确对比 |
| JSON嵌套展开 | 子表数据逐层对比 |
| 记录数量 | 列表型数据条数对比 |
| 数据新鲜度 | 实时更新时间 vs 离线数据时间 |
