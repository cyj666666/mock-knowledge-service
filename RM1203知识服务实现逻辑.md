# RM1203 知识服务实现逻辑

> 29个moduleCode → 实时调用630生产接口 → 统一返参

---

## 一、调用入口

```
POST /api/knowledge
Content-Type: application/json

{
  "transcode": "RM1203",
  "account": "POC",
  "source": "EDS",
  "userid": "EDS",
  "orgid": "EDS",
  "params": {
    "entName": "宁波隆源股份有限公司",
    "moduleCode": "BasicProfile-EnterpriseInfo-630API"
  }
}
```

入口在 `KnowledgeController.java`，transcode=`RM1203` 时分发到 `RealtimeKnowledgeService.handle()`。

## 二、核心链路

```
┌─────────────────┐
│ RM1203 请求      │
│ entName + code   │
└────────┬────────┘
         ▼
┌─────────────────────────────┐
│ RealtimeKnowledgeService    │
│ 1. 查 rm1203_modulecode_    │
│    mapping.json → 配置      │
│ 2. 有 preStep? → 二步调用   │
│ 3. 调 630 生产接口           │
│ 4. 提取 dataKey 子表         │
│ 5. 包装 KnowledgeResponse   │
└────────┬────────────────────┘
         ▼
┌─────────────────┐     ┌──────────────────┐
│ RealtimeApiClient│ ──▶ │ 630 生产环境      │
│ (HttpURLConn)   │ ◀── │ hub.amardata.com │
└─────────────────┘     └──────────────────┘
```

## 三、配置文件：rm1203_modulecode_mapping.json

位置：`offline-data/rm1203_modulecode_mapping.json`

每条 moduleCode 的配置结构：

```json
{
  "BasicProfile-EnterpriseInfo-630API": {
    "name": "企业工商查询服务",
    "transcode": "R11103V3",
    "dataKey": null,
    "preStep": null,
    "extraParams": null
  }
}
```

### 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|:---:|------|------|
| `name` | ✅ | 中文名称 | "企业工商查询服务" |
| `transcode` | ✅ | 630接口编号 | "R11103V3" |
| `dataKey` | ❌ | 从返回data中提取的子表key。null=整包返回 | "basicList" |
| `preStep` | ❌ | 二步调用配置，见下文 | |
| `extraParams` | ❌ | 额外固定参数，合并到请求的params中 | {"pageIndex": 1} |

## 四、三种调用模式

### 模式1：直调（无 dataKey，无 preStep）

```json
{
  "transcode": "R11C72",
  "dataKey": null,
  "preStep": null
}
```

逻辑：`call(R11C72, entName, "1", {})` → 返回完整 data 数组。

**适用接口**：R11C72、RA403、R227V2、R217V2、R241V2、R243V2、R1212V2、R1201V3、R705V5、R354V2、RA1216、R11101、R1199、R4G04V2

---

### 模式2：直调 + 子表提取（有 dataKey）

```json
{
  "transcode": "R11103V3",
  "dataKey": "basicList",
  "preStep": null
}
```

逻辑：`call(R11103V3, ...)` → 从 `data[0]` 中只取 `data[0].basicList` 返回。

**适用接口**：R11103V3 的各子表拆分（原设计44个code时用，现29个code中用 null 返回整包）

---

### 模式3：二步调用（有 preStep）

```json
{
  "transcode": "R11Z03",
  "preStep": {
    "transcode": "R11103V3",
    "dataKey": "basicList",
    "extractField": "creditCode",
    "useAsParam": "name"
  }
}
```

逻辑：
1. 先用企业名调 `R11103V3` → 从 `data[0].basicList[0].creditCode` 提取统一信用代码
2. 用提取到的信用代码调 `R11Z03`

**适用接口**：R11Z03（需信用代码）、R1167（需信用代码）、R11133（需信用代码）、R1618（需信用代码）、R255（需信用代码）、R4G06V2（需R4G05的serialNo）、Financial-*（需R4G05的serialNo）

### preStep 配置字段

| 字段 | 说明 |
|------|------|
| `transcode` | 前一步调哪个630接口 |
| `dataKey` | 从前一步返回中提取哪个子表 |
| `extractField` | 提取的字段名 |
| `useAsParam` | 注入到主调用的参数名 |

| 接口 | 参数名 | 说明 |
|------|--------|------|
| R11C72 | `entname` | |
| RA403 | `companyName` | |
| R705V5 | `TaxPayerName` | |
| R11101/R1199 | `applicantName` | |
| R4G04V2 | `name` + `nameType` | |
| R227V2 | `entname` | |
| R354V2 | `entName` | |
| R301V2 | `EntName` | 大写E |
| R1167 | `entName` | |
| 其它 | `name` | 默认 |

## 六、参数适配（extraParams）

有些接口需要固定额外参数，通过 `extraParams` 配置：

```json
{
  "transcode": "R11101",
  "extraParams": {"pageIndex": 1}
}
```

或：

```json
{
  "transcode": "R217V2",
  "extraParams": {"NameType": "1", "isRelationHisName": "true"}
}
```

## 八、如何新增 moduleCode

1. 在 `rm1203_modulecode_mapping.json` 加一条：

```json
"NewModule-630API": {
  "name": "新模块名",
  "transcode": "Rxxxxx",
  "preStep": null,
  "extraParams": {"pageIndex": 1}
}
```

2. 如果新接口参数名不同，在 `RealtimeApiClient.adjustParams()` 的 switch 中加 case：

```java
case "Rxxxxx": params.put("yourParamName", entName); break;
```

3. 重启服务即生效。
