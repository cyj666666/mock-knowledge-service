# 知识服务挡板 — 部署说明

> 版本 1.0.0 | 2026-06-08

---

## 一、物料清单

解压后的目录结构：

```
mock-knowledge/
├── mock-knowledge-service.jar    # Spring Boot 可执行 jar（JDK 8）
├── startup.sh                    # 启动脚本
├── shutdown.sh                   # 停止脚本
├── offline-data/                 # 离线数据包（已内置数据 + keymap）
│   ├── keymap.json
│   └── {企业名}/
│       └── {moduleCode}/
│           ├── RM1201.json
│           └── RM1202.json
└── 部署说明.md                    # 本文档
```

---

## 二、环境要求

| 要求 | 说明 |
|------|------|
| JDK | 1.8 或以上 |
| 操作系统 | Linux（CentOS 7+ / Ubuntu 18+） |
| 端口 | 默认 18080（可配置） |
| 磁盘 | jar 18MB + 数据包空间 |

---

## 三、部署步骤

### 3.1 上传并解压

```bash
# 上传压缩包到服务器
scp mock-knowledge.tar.gz root@服务器IP:/opt/app/

# 解压
cd /opt/app
tar -xzf mock-knowledge.tar.gz
cd mock-knowledge
```

### 3.2 启动

```bash
chmod +x startup.sh shutdown.sh
./startup.sh
```

### 3.3 验证

```bash
# 健康检查
curl -X POST http://localhost:18080/api/knowledge \
  -H "Content-Type: application/json" \
  -d '{"transcode":"RM1201","params":{"entName":"test","moduleCode":"test"}}'

# 期望返回: {"code":"9999","msg":"无匹配数据","data":[]}
# 返回 9999 说明服务正常运行
```

## 四、运维命令

| 操作 | 命令 |
|------|------|
| 启动 | `./startup.sh` |
| 停止 | `./shutdown.sh` |
| 重启 | `./shutdown.sh && ./startup.sh` |
| 查看日志 | `tail -f logs/mock-knowledge-service.log` |
| 查看启动日志 | `tail -f logs/console.log` |
| 确认进程 | `ps -ef \| grep mock-knowledge-service.jar \| grep -v grep` |

---

## 五、配置修改

直接编辑 `startup.sh` 顶部的配置变量，保存后 `./shutdown.sh && ./startup.sh` 重启生效：

```bash
# startup.sh 顶部
SERVER_PORT=18080                    # 改端口
DATA_PATH="./offline-data"           # 改数据路径
JAVA_OPTS="-Xms256m -Xmx512m -Dfile.encoding=UTF-8"  # 改内存/编码
```

无需手动拼命令行参数。

---

## 六、更新数据

解压后的 `offline-data/` 目录可直接修改，**无需重启服务**：

- 新增数据：按目录结构放入文件
- 删除数据：删除对应目录或文件
- 新增了内容分析类数据后，需同步更新 `offline-data/keymap.json`

---

## 七、日志说明

| 日志文件 | 内容 |
|----------|------|
| `logs/mock-knowledge-service.log` | 业务日志（请求记录、匹配结果） |
| `logs/console.log` | 控制台输出（JVM 信息、异常堆栈） |

日志按 10MB 自动滚动，保留最近 30 天。

---

## 八、常见问题

| 问题 | 解决 |
|------|------|
| 启动失败 | 查看 `logs/console.log` 尾部 20 行 |
| 返回 9999 | 确认 `offline-data/{企业名}/{moduleCode}/RM1201.json` 存在且为合法 JSON |
| 端口被占用 | 修改 `startup.sh`，加 `--server.port=18081` |
| 中文乱码 | 确认 JSON 文件为 UTF-8 编码 |
